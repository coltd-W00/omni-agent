### D-008: Session ID và resume mechanism đã confirmed (RESOLVED Open Q1 & Q2)
- **Claude:** Session ID lấy từ `session_id` field trong JSON output. Resume bằng `claude --continue --session-id <uuid>` với comment làm prompt.
- **Codex:** Session ID là UUID. Capture ưu tiên từ JSON/event output khi start; fallback scan `~/.codex/sessions/` lọc theo cwd + modified time gần nhất. Resume bằng `codex resume <uuid>` hoặc `codex exec resume <uuid>`.
- **Chuẩn hóa nội bộ:** Session ID lưu dưới dạng UUID string, không phụ thuộc format của từng CLI.
- **Nguồn:** Loc xác nhận từ docs của Codex và Claude CLI.
- **Ảnh hưởng:** FR-5 và FR-7 đã cập nhật. Open Questions 1 & 2 đóng lại.

### D-009: Subprocess lifecycle — backend là process owner
- **Quyết định:** Đóng browser/tab không kill subprocess. Backend giữ subprocess sống.
- **Kill conditions:** (1) user bấm Stop/Cancel, (2) backend shutdown, (3) timeout policy.
- **Backend shutdown:** flush trạng thái Running → Paused xuống DB trước khi exit để đảm bảo resume được sau.
- **Nguồn:** Loc xác nhận trực tiếp.
- **Ảnh hưởng:** FR-6 đã cập nhật. Open Question 3 đóng lại.

### D-010: Multi-project và log storage strategy
- **Multi-project:** MVP hỗ trợ nhiều Project. Task Board lọc theo Project active. Xóa Project có Task bị block.
- **Log storage:** Full output ghi ra file trên disk (path lưu trong DB). DB chỉ lưu metadata + tail (last ~100 lines / 10 KB). UI hiển thị tail mặc định, có nút "View full log" và download raw log. UI không render full raw output.
- **Nguồn:** Loc xác nhận trực tiếp.
- **Ảnh hưởng:** FR-0 thêm mới, FR-1 cập nhật, FR-8 cập nhật. Open Questions 4 & 5 đóng lại. Tất cả Open Questions đã resolved.

### D-011: Task ID scope, Resume retry, Status simplification
- **Task ID:** Scope theo Project, format `{PROJECT_KEY}-NNN`. Project có thêm field "key" (unique, uppercase).
- **Resume:** Comment là tùy chọn. Retry không cần comment — app ghi "retry" vào Run log.
- **Status MVP:** Bỏ `Needs Input` và `Needs Review` khỏi transitions tự động. MVP chỉ cần 8 trạng thái: Draft, Ready, Assigned, Running, Paused, Failed, Done, Cancelled. Người dùng tự Mark Done khi xong. `Needs Input`/`Needs Review` có thể thêm lại ở v2 với output parsing.
- **Nguồn:** Loc xác nhận trực tiếp.
